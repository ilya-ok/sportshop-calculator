# Админка и мультисайт

## Таблица калькуляторов

Колонки: Название | Категория | Хук | Шорткод | Действия.

В ячейке "Название" под именем — тип калькулятора (`<small>Газон</small>` / `<small style="color:#0073aa">Линолеум</small>`).

В ячейке "Действия" — кнопки: **Изменить**, **Удалить**, **Сброс кеша** (+ **Синхр.** на мультисайте). Кнопка "Сброс кеша" использует класс `.ssc-clear-cache-btn`, обработчик — делегированный `$(document).on('click', '.ssc-clear-cache-btn', ...)`, очищает весь transient `ssc_calc_cache` через `ssc_clear_cache` AJAX.

## Форма редактирования — поля по типу

Тип переключается через `#ssc-calc-type`. При смене:
- `.ssc-grass-only` — видны только для газона (класс управляется JS).
- `.ssc-linoleum-only` — видны только для линолеума (напр., поле цены краски).
- `.ssc-grass-label` / `.ssc-linoleum-label` — двойные подписи для одного поля (напр., "Клей: цена за банку" / "Клей (напольный): цена за кг").

### Поля для линолеума
- **Клей** (`#ssc-glue-price`) — цена за кг напольного клея
- **Сварочный шнур** (`#ssc-tape-price`) — цена за м.п.
- **Разметка поля** (`#ssc-markup-enabled`) — включает выбор Волейбол/Баскетбол/Мини-футбол
- **Краска для разметки** (`#ssc-paint-price`, `.ssc-linoleum-only`) — цена за банку, сохраняется в `paint_price`

## Изображения canvas (class-ssc-admin.php)

Для каждой подкатегории рендерится несколько полей ввода изображений (для `none`, `football`, `mini-football`, `tennis`, `hockey`). Данные сохраняются во вложенной структуре `canvas_images[subcategory][markup_type]`.

`admin.js` слушает изменения в textarea `subcategory_slugs`. При каждом изменении парсит slug'и по строкам и динамически рендерит для каждого: label с именем подкатегории, input URL, кнопка выбора из медиа-библиотеки, превью. Поля появляются в блоке `.ssc-canvas-images-wrap`.

## Мультисайт (Синхронизация)

Доступна только если `is_multisite()`. Три точки входа:

### 1. Кнопка в строке таблицы — `.ssc-sync-row-btn`
- Появляется в колонке "Действия" рядом с "Изменить"/"Удалить" для каждого калькулятора.
- AJAX: `ssc_sync_calculator`, передаёт `calc_id`.
- После успеха показывает "✓ N гор.", через 2 сек возвращается к "Синхр.".

### 2. Кнопка "Синхронизировать все" — `#ssc-sync-all-btn`
- В шапке страницы рядом с "+ Добавить".
- AJAX: `ssc_sync_all_calculators` — берёт **все** калькуляторы текущего сайта и копирует/обновляет их на всех остальных сайтах сети.
- Калькуляторы на целевом сайте, которых нет на текущем, остаются нетронутыми.

### 3. Кнопка в форме редактирования — `#ssc-sync-btn`
- В footer панели редактирования (исторически первая кнопка).
- AJAX: `ssc_sync_calculator`, берёт ID из `#ssc-calc-id`.

### Логика AJAX-обработчиков
- `ajax_sync_calculator()` — один калькулятор → все сайты. Создаёт или обновляет запись с тем же ID.
- `ajax_sync_all_calculators()` — все калькуляторы → все сайты. Мерджит поверх существующих (не удаляет чужие).
- Оба используют `switch_to_blog()` / `restore_current_blog()`.
- Nonce: `ssc_admin_nonce`.
